# Оценка состояния репозитория PhysicsLab
## Дата: 17 мая 2026 г.

---

## 📋 Резюме

PhysicsLab находится в **этапе подготовки (Phase 0)** с хорошо структурированной документацией и прочной архитектурой Astro. Проект **не готов к production**, что явно указано в README. Однако он имеет достаточно потенциала для перехода к Phase 1 (инвентаризация контента) при условии решения критических проблем.

**Общая оценка: 6,5/10**

---

## ✅ СИЛЬНЫЕ СТОРОНЫ

### 1. Архитектура и организация кода (9/10)

**Чистое разделение компонентов:**
- `src/components/ui/` — базовые компоненты (Badge, Button, SurfaceCard, SectionHeader)
- `src/components/learning/` — обучающие компоненты (FormulaCard, QuizCard, MisconceptionCard, WorkedExample, SimulationPanel, CharacterBubble, ExamBlock)
- `src/components/simulations/` — интерактивные лаборатории (UniformMotionLab)
- `src/components/math/` — математические компоненты (MathFormula)
- `src/components/playground/` — экспериментальное пространство

**Цифры:**
- 18 Astro-компонентов
- 2 TypeScript модуля
- 3 Astro layout'а (BaseLayout, ChapterLayout)
- Чистый и расширяемый граф зависимостей

**Положительные точки:**
- Каждый компонент имеет чёткую ответственность
- Пропсы типизированы
- Нет кругозависимостей
- Props-based архитектура готова для MDX интеграции

### 2. Дизайн-система (8/10)

**Полный набор токенов в `src/styles/tokens.css`:**
- Цветовая палитра: фон, поверхности, границы, текст (разные уровни контраста)
- Специальные цвета для физических величин:
  - `--pl-position: #58d8ff` (cyan)
  - `--pl-velocity: #ffd84d` (желтый)
  - `--pl-time: #6ee7a8` (зелёный)
  - `--pl-acceleration: #ff8a5b` (оранжевый)
- Пространство: `--pl-space-*` (4px до 80px)
- Радиусы и тени стандартизированы
- Dark theme готов (`color-scheme: dark`)

**Проблема:** Токены описаны, но не везде применяются в legacy-коде.

### 3. Тестирование (7/10)

**Physics тесты:**
- `npm run test:physics` → ✅ проходит
- `tests/physics/kinematics.test.mjs` тестирует базовые функции
- Unit-тесты для кинематики (равномерное движение)

**E2E тесты (Playwright):**
- `npm run test:e2e` — полный Playwright suite
- `npm run screenshots` — автоматизация скриншотов
- Проверяется 5 viewport'ов:
  - Desktop: 1440×1000
  - Laptop: 1280×900
  - Tablet: 768×1024
  - Mobile: 390×844
  - Narrow Mobile: 360×800
- Для каждого viewport'а генерируются:
  - Full-page скриншоты
  - Hero-секция
  - Chapter-demo блок

**Проверки качества:**
- Горизонтальный скролл не допускается
- Видимость элементов проверяется
- Интерактивность симуляции проверяется (play/pause кнопки)

### 4. Документация (9/10)

**Объём и качество (5000+ строк):**
- `docs/roadmap.md` (277 строк) — дорожная карта до Phase 4
- `docs/content-rules.md` (265 строк) — правила контента и оригинальности
- `docs/design-and-learning-rules.md` (298 строк) — педагогические принципы
- `docs/technical-migration-plan.md` (862 строк) — архитектурный план миграции
- `docs/asset-policy.md` — политика ассетов
- `docs/agent-instructions.md` — инструкции для разработчиков/агентов

**Ясная иерархия:**
- Legacy (старый HTML-прототип)
- Prototype (дизайн-исследования)
- Foundation (новая Astro база)

**Никаких двусмысленностей:** всё явно описано, что готово, что в процессе, что требует проверки.

### 5. TypeScript (7/10)

- `tsconfig.json` использует `astro/tsconfigs/strict` (хороший базис)
- Path alias `@/*` для удобства при импортах
- Типизированы основные интерфейсы:
  - `FormulaVariable` (символ, метка, единицы, тон)
  - `MissionItem` (метка, статус)
  - `UniformMotionSeriesParams` и related типы
  - Параметры компонентов

**Недостаток:** Нет типизации для content schema (нужна интеграция с Content Collections).

### 6. Живой стиль и идентичность (8/10)

- Выбран визуальный стиль: **Dark Anime Lab**
- Брендовые элементы: персонажи, диалоги, живой тон в UI
- Не копирует corporate EdTech (как требуют rules)
- Понятный визуальный язык для школьников

## ⚠️ ПРОБЛЕМЫ И ОГРАНИЧЕНИЯ

### 1. 🔴 КРИТИЧЕСКОЕ: Несовместимость Node.js (блокирует build)

**Текущая среда:**
- Node.js: v20.20.2
- npm: 10.8.2

**Требуемая версия (Astro 6.3.3):**
- Node.js: >=22.12.0
- npm: >=9.6.5

**Результат:**
```
> npm run build
Node.js v20.20.2 is not supported by Astro!
Please upgrade Node.js to a supported version: ">=22.12.0"
```

**Влияние:** 🟡 Не критично для локальной разработки, 🔴 критично для CI/CD pipeline.

---

### 2. 🟡 БЕЗОПАСНОСТЬ: 5 уязвимостей YAML (средне-критично)

**Проблема:** Stack overflow в YAML при глубокой вложенности коллекций

**Цепь зависимостей:**
```
yaml (2.0.0 - 2.8.2)
  ↓
yaml-language-server (1.11.1 - 1.21.1)
  ↓
volar-service-yaml (≤0.0.70)
  ↓
@astrojs/language-server (≥2.14.0)
  ↓
@astrojs/check (0.9.9) ← CURRENT
```

**Решение:**
```bash
npm audit fix --force
# Обновит @astrojs/check с 0.9.9 до 0.9.2 (breaking change)
```

**Влияние:** 
- 🟢 На production-build (static output) — нет влияния
- 🟡 На CI/CD pipeline — нужно адресировать
- 🟡 На IDE tooling (@astrojs/check) — может повлиять

---

### 3. 🟡 АРХИТЕКТУРНЫЙ ДОЛГ: UniformMotionLab слишком большой

**Размер:** 1398 строк в одном файле

**Состав:**
- 200+ строк frontmatter логики (вычисления, state)
- 400+ строк inline-стилей (много rgba значений)
- 400+ строк SVG графиков
- 300+ строк форм и элементов управления

**Проблемы:**
- Сложно читать и поддерживать
- Сложно тестировать отдельные части
- Сложно переиспользовать компоненты (например, GraphCanvas)
- При добавлении новых симуляций код будет расти линейно

**Рекомендация:** Разбить на подкомпоненты:
1. `GraphCanvas.astro` (SVG-граф, ~200 строк)
2. `ControlPanel.astro` (слайдеры и кнопки, ~150 строк)
3. `ValueDisplay.astro` (отображение текущих значений, ~100 строк)
4. `PlaybackControls.astro` (play/pause/reset, ~100 строк)
5. `UniformMotionLab.astro` (оркестратор, ~200 строк)

---

### 4. 🔴 АРХИТЕКТУРНЫЙ ДОЛГ: Content Model не реализована

**Что обещано в документации:**
- Astro Content Collections
- MDX frontmatter
- Типизированная схема для глав, задач, квизов

**Что есть на самом деле:**
- Контент hardcode'ен в пропсах Astro-компонентов
- Пример (`src/pages/lab-preview.astro:27-31`):
  ```astro
  const formulaVariables: FormulaVariable[] = [
    { symbol: "x, x0", label: "координата", units: "м", tone: "position" },
    { symbol: "v", label: "скорость", units: "м/с", tone: "velocity" },
    { symbol: "t", label: "время", units: "с", tone: "time" }
  ];
  ```

**Проблемы:**
- При добавлении 20+ глав — беспорядок в коде
- Нет переиспользования контента
- Сложно версионировать контент отдельно от кода
- Нет валидации при публикации

**Необходимое действие:** Перейти на Content Collections до Phase 1.

---

### 5. 🟡 ОГРАНИЧЕННЫЙ ОХВАТ ФИЗИКИ

**Реализовано:**
- Только равномерное движение (uniform motion)
- Библиотека: `src/lib/physics/kinematics.ts` (126 строк)
- Функции: `uniformPosition()`, `uniformVelocity()`, `clampNumber()`, направление

**Требуется по документации (roadmap):**
- Графики: x(t), v(t), a(t), все типы
- Виды движения: равномерное, равноускоренное, замедленное, круговое
- Кинематика: v = v0 + at, x = v0*t + (a*t²)/2
- Динамика: F = ma, инерция, трение
- Работа и энергия: W = F*s, E_k, E_p, закон сохранения
- МКТ и термодинамика

**Тестирование:**
- ✅ Unit-тесты для кинематики проходят
- ⚠️ Интеграционные тесты только для UI (screenshot'ы)
- ❌ Нет unit-тестов для других физических модулей

---

### 6. 🟡 НЕПОЛНОТА ДОСТУПНОСТИ (a11y)

**Что есть:**
- ✅ `role="main"`, `aria-labelledby`, `aria-label` в некоторых местах
- ✅ `aria-live` для динамических обновлений
- ✅ `focus-visible` для клавиатуры
- ✅ Проверка `prefers-reduced-motion`
- ✅ Семантический HTML (много `<section>`, `<article>`)

**Что отсутствует:**
- ❌ MathML или aria-описания для формул (KaTeX только визуально)
- ❌ Полная проверка contrast ratio в dark theme
- ❌ Семантика для radio-groups в legacy (`mechanics-menu.html`)
- ❌ Текстовые альтернативы для SVG графиков
- ❌ Полная поддержка screen readers для интерактивных элементов

**Статус:** 🟡 Базовая a11y есть, но не production-уровня.

---

### 7. 🟡 LEGACY И FOUNDATION НЕ СИНХРОНИЗИРОВАНЫ

**Legacy CSS (`css/style.css`):**
- Старая система токенов
- CDN Tailwind
- Работает в браузере, но не интегрирована с новой дизайн-системой

**Foundation CSS (`src/styles/tokens.css`):**
- Новая система токенов
- Не используется в legacy-страницах

**Риск:** При разработке двух параллельных систем дизайна будут расхождения и путаница.

**Рекомендация:** На Phase 1 выбрать единую дизайн-систему и мигрировать legacy.
