# PhysicsLab Content Rules

Этот документ задает правила для учебного текста, задач, квизов, формул, графиков, источников и статусов проверки.

Источник правил: `PhysicsLab_research_report.pdf`, версия от 15 мая 2026.

## 1. Content Mission

PhysicsLab объясняет школьную физику для Беларуси ясно, живо, экзаменационно осознанно и физически корректно.

Контент должен:

- помогать понять физический смысл, а не только запомнить формулы;
- тренировать реальные навыки решения задач;
- учитывать формат ЦЭ/ЦТ;
- снижать тревожность;
- сохранять живой мемно-аниме голос проекта;
- быть оригинальным и проверяемым.

## 2. Source Hierarchy

Приоритет источников:

1. Официальные программы Беларуси, НИО, ADU, Министерство образования.
2. РИКЗ: спецификации, памятки, демонстрационные и экзаменационные материалы.
3. Peer-reviewed learning science и physics education research.
4. Надежные открытые образовательные ресурсы.
5. Авторские объяснения, схемы, задачи и симуляции PhysicsLab.

Если официальное требование не проверено, не придумывай его. Добавь `TODO` или статус проверки.

## 3. Belarus and Exam Context

PhysicsLab ориентируется на школьников Беларуси и должен учитывать:

- программу VII-IX и X-XI классов;
- формат ЦЭ/ЦТ;
- задания части A: закрытый ответ, один или несколько вариантов;
- задания части B: численный ответ;
- важность механики, электродинамики, МКТ и термодинамики;
- навыки чтения графиков, схем, таблиц и рисунков;
- перевод единиц и округление.

Данные о спецификациях экзамена со временем могут меняться. При обновлении экзаменационного контента сверяйся с актуальными официальными источниками.

## 4. Originality Rule

Нельзя копировать:

- абзацы из учебников;
- задачи из учебников, сборников, РИКЗ или ЦЭ/ЦТ;
- формулировки с минимальной заменой чисел;
- порядок шагов и сюжет задачи, если он узнаваемо повторяет источник;
- скриншоты и иллюстрации из учебных материалов без прав.

Можно:

- использовать физические законы, формулы и общеизвестные факты;
- опираться на структуру официальной программы;
- брать проверяемый навык и экзаменационный формат;
- писать собственное объяснение;
- создавать собственный сюжет, числа, схему и решение.

Оригинальная задача должна отличаться не только числами, но и конкретным контекстом, формулировкой и ходом предъявления.

## 5. Review Status

Новый физический контент не считается готовым без статуса проверки.

Используй один из вариантов:

```yaml
authorReviewRequired: true
```

или

```yaml
reviewStatus: "needs-physics-review"
```

Для будущей структурированной модели можно использовать:

```yaml
review:
  status: "needs-physics-review"
  reviewer: null
  reviewedAt: null
  notes: []
```

Статус нужен для:

- глав;
- задач;
- квизов;
- карточек повторения;
- симуляций;
- формул и таблиц величин;
- экзаменационных блоков.

## 6. Formula Rules

Формула никогда не появляется одна.

Рядом с формулой должны быть:

- смысл словами;
- список переменных;
- единицы СИ;
- условия применимости;
- когда формулу использовать нельзя;
- короткий пример;
- типичная ошибка.

Мини-шаблон:

```yaml
formula: "s = vt"
meaning: "Путь при равномерном движении равен скорости, умноженной на время."
variables:
  s: "путь, м"
  v: "скорость, м/с"
  t: "время, с"
conditions:
  - "скорость постоянна"
  - "движение рассматривается вдоль одной линии или по длине траектории"
notFor:
  - "движение с ускорением"
  - "задачи, где нужна координата с учетом направления"
commonMistake: "Использовать s = vt, когда скорость меняется."
authorReviewRequired: true
```

## 7. Graph Rules

Каждый график должен иметь:

- подписи осей;
- единицы;
- объяснение наклона;
- объяснение площади, если она имеет физический смысл;
- типичное неверное чтение;
- check question.

Не предполагай, что ученик сам понимает графики.

Для кинематики обязательно различать:

- график как зависимость величины от времени;
- траекторию как путь тела в пространстве.

## 8. Task Metadata

Каждая новая задача должна иметь метаданные.

Минимальный набор:

```yaml
id: "mech-kin-03-012"
topic: "Равноускоренное движение"
skill:
  - "Выбор формулы"
  - "Подстановка в СИ"
difficulty: 2
format: "B"
answerType: "integer"
units: "м"
requiresRounding: true
misconceptions:
  - "Использует формулу равномерного движения"
source: "original"
authorReviewRequired: true
```

Решение должно содержать:

1. Дано.
2. Перевод единиц.
3. Схему или модель, если нужна.
4. Выбор закона.
5. Подстановку.
6. Ответ.
7. Проверку единиц.
8. Проверку физической правдоподобности.

## 9. Quiz Rules

В мини-квизе смешивай типы вопросов:

- recall;
- concept check;
- graph check;
- misconception check;
- worked-step;
- unit check;
- exam A;
- exam B;
- transfer.

Каждый неверный ответ получает feedback:

1. Что именно не так.
2. Почему это типичная ошибка.
3. Какой принцип нужен.
4. Следующий короткий шаг.

Не использовать feedback вида "неверно" без объяснения.

## 10. Chapter Frontmatter

Рекомендуемый frontmatter будущей главы:

```yaml
title: "Равноускоренное движение"
module: "Механика"
grade: 9
examWeight: "Высокий"
difficulty: 3
estimatedTime: "25-35 минут"
prerequisites:
  - "Скорость"
  - "Графики x(t) и v(t)"
  - "Единицы СИ"
skills:
  - "Понимать ускорение"
  - "Читать график v(t)"
  - "Решать задачи на равноускоренное движение"
examFormats:
  - "Часть A: выбор ответа"
  - "Часть B: численный ответ"
authorReviewRequired: true
```

## 11. Tone Rules

Тон:

- дружелюбный;
- прямой;
- подростково-живой;
- немного мемный;
- без унижения;
- без фальшивой мотивационной болтовни.

Хороший текст объясняет, почему ошибка логична, и помогает выбраться из нее.

Плохой текст стыдит ученика или говорит "просто запомни".

## 12. Copyright Rules for Content

Не копировать учебники, сборники, РИКЗ, чужие картинки, кадры аниме и мемы без лицензии.

Факты, формулы и идеи можно использовать. Конкретное выражение: текст, иллюстрация, задача, персонаж, кадр, дизайн - обычно защищается.

Для каждого внешнего источника в учебном контенте фиксируй:

- что использовано;
- зачем использовано;
- ссылка;
- надежность;
- дата проверки;
- ограничения.

Если есть сомнение по правам, не добавляй материал.
