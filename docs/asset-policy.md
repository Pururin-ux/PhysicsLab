# PhysicsLab Asset Policy

Этот документ задает правила для изображений, персонажей, мемов, схем, иконок, аудио, видео и сгенерированных ассетов.

Это рабочая политика снижения риска, не юридическая консультация.

## 1. Core Rule

Нельзя копировать чужие картинки, кадры аниме, известных персонажей, мемы, скриншоты учебников, сборников и материалов РИКЗ без лицензии.

Идеи, факты, формулы и методы можно использовать. Конкретное выражение обычно защищается: текст, иллюстрация, задача, кадр, персонаж, дизайн, фотография.

## 2. Allowed Assets

Можно использовать:

- собственные схемы PhysicsLab;
- собственных оригинальных персонажей;
- собственные фотографии и рисунки;
- ассеты public domain;
- CC0;
- CC BY при корректной атрибуции;
- CC BY-SA при соблюдении условий;
- MIT/Apache/BSD-иконки при соблюдении лицензии;
- сгенерированные ассеты, если права и условия сервиса позволяют использование в проекте.

Для каждого внешнего ассета нужна запись о лицензии.

## 3. Forbidden Assets

Нельзя использовать:

- кадры из аниме без лицензии;
- известных аниме-персонажей;
- чужие мем-картинки без понятной лицензии;
- скриншоты учебников и сборников;
- изображения из заданий РИКЗ;
- фотографии из Google Images без проверки прав;
- "почти такого же" персонажа, узнаваемо основанного на существующей IP;
- логотипы и бренды без необходимости и прав;
- ассеты с непонятным происхождением.

Если происхождение неясно, ассет не добавлять.

## 4. Anime and Meme Style

Аниме/мемная эстетика разрешена как стиль, но не как копирование.

Правильно:

- оригинальный персонаж в аниме-вдохновленной стилистике;
- мемный текст без чужой картинки;
- собственная реакционная иллюстрация;
- оригинальная мини-комикс-сцена, которая объясняет ошибку.

Неправильно:

- вставить кадр из аниме;
- перерисовать узнаваемого персонажа;
- взять популярный мем-шаблон без лицензии;
- использовать чужую картинку как "ну это же мем".

## 5. Asset Register

В будущей структуре проекта рекомендуется вести `assets/LICENSES.md` или аналогичный реестр лицензий.

Минимальный шаблон:

```markdown
## characters/mika-neutral.png

- Author: PhysicsLab / project author
- Source: original
- License: Project-owned
- Date added: 2026-05-15
- Notes: Original character, not based on existing anime IP.
```

Для внешнего ассета:

```markdown
## icons/check.svg

- Author: Vendor or creator name
- Source: https://example.com/icon
- License: MIT / CC BY 4.0 / other
- Attribution: required text if any
- Date added: YYYY-MM-DD
- Notes: Used for UI status icon.
```

## 6. Generated Assets

Сгенерированные изображения разрешены, если:

- prompt не просит скопировать конкретного художника, кадр, персонажа или мем;
- результат не похож на существующую IP;
- лицензия инструмента допускает использование;
- ассет помогает обучению или атмосфере PhysicsLab;
- файл внесен в asset register.

Для персонажей указывай:

- роль персонажа;
- что он объясняет;
- почему нужен;
- подтверждение, что это оригинальный персонаж.

## 7. Diagrams and Physics Images

Учебные схемы лучше создавать самим.

Требования:

- схема должна быть физически корректной;
- подписи величин и единиц читаемы;
- направление векторов понятно;
- цвет не единственный носитель смысла;
- есть alt-текст или текстовое пояснение;
- новый физический смысл имеет `authorReviewRequired` или `needs-physics-review`.

## 8. File Naming

Рекомендуемый стиль:

- `characters/mika-neutral.png`;
- `characters/mika-warning.png`;
- `diagrams/mechanics/free-body-block.svg`;
- `diagrams/optics/convex-lens-rays.svg`;
- `memes/original/velocity-graph-trap.png`;
- `ui/icons/check.svg`.

Не использовать имена:

- `image1.png`;
- `anime.png`;
- `meme-final-final.png`;
- `naruto-like-character.png`;
- `rikz-screenshot.png`.

## 9. Accessibility

Для смысловых изображений нужен alt-текст:

- что изображено;
- какая физическая идея передается;
- какие обозначения важны.

Декоративные изображения должны быть скрыты от screen reader.

Мемы не должны передавать единственную важную информацию. Если мем объясняет ловушку, рядом должен быть обычный текст.

## 10. Review Before Adding an Asset

Перед добавлением проверь:

- есть ли право использовать ассет;
- зафиксирована ли лицензия;
- нужен ли ассет для обучения;
- не перегружает ли он экран;
- не отвлекает ли от формулы, графика или задачи;
- есть ли alt-текст;
- не нарушает ли он стиль PhysicsLab;
- не похож ли он на защищенного персонажа или кадр.

Если любой пункт вызывает сомнение, не добавляй ассет без решения владельца проекта.
