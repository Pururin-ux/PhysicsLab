# Reference Solution Pilots v1

Дата проверки: 2026-07-13
Baseline: `d1d562f609d2b6270af3ec8590dd6e14deb830ab`

Документ описывает шесть авторских учебных примеров для проверки reference-solution framework. Формулировки и разборы написаны для PhysicsLab и не скопированы из РИКЗ, образовательных платформ или конкурентных интерфейсов. Примеры не обозначаются как официальные задания ЦТ/ЦЭ.

## Намеренные ограничения

- `ohm-law` не разбирает последовательные и параллельные соединения.
- `vt-area` не обучает определению ускорения по наклону; это отдельное семейство.
- `friction-force` не включает наклонную плоскость и трение покоя.
- `average-speed-segments` не рассматривает остановки и разные системы единиц.
- `heat-balance-simple` ограничен одним веществом без фазового перехода и потерь.
- `thin-lens-image-distance` использует одну однозначную знаковую convention для действительного предмета и изображения; рассеивающая линза и мнимое изображение вне scope.

## Framework evaluation

Одна структура «Модель → Закон → Подстановка → Ответ» подошла всем шести пилотам. Она не заставляет графическую задачу притворяться обычной формульной: в `vt-area` модель сначала связывает перемещение с площадью, а закон описывает геометрию трапеции. Для линзы модель и статическая схема фиксируют расположение предмета и изображения до вычислений.

Optional `check` оказался содержательно полезен для закона Ома, графика, средней скорости, теплового баланса и линзы. Для трения отдельная проверка не добавлена: размерность и последовательность N = mg, Fтр = μN уже видны в основных шагах, а дополнительный блок только повторил бы решение.

Graph и optics discriminants достаточны для пилота. Они используют существующие `PhysicsGraphSpec` и `OpticsDiagramSpec`, поэтому данные остаются сериализуемыми, а визуалы получают существующее SVG- и accessibility-поведение. Перед заполнением остальных 29 семейств стоит проверить, нужны ли отдельные visual variants для электрических схем и векторных диаграмм; добавлять их заранее не следует.

Главный риск formula-only teaching проявился в `vt-area`, `average-speed-segments` и `heat-balance-simple`: формула без модели не объясняет, почему берётся площадь, почему нельзя усреднять скорости и почему температуры взвешиваются массами. Поэтому поля `model.explanation` и `typicalMistake` должны оставаться обязательными.

Framework пригоден для дальнейшей ручной редакционной работы, но ещё не доказан для массового заполнения всех 29 семейств. До масштабирования нужны: методическая проверка новых визуальных discriminants, единый словарь знаковых convention и content-review процесс с числовыми residual tests. Автоматически генерировать reference solutions из task blueprint нельзя: генератор гарантирует варианты задач, но не качество учебного рассуждения.
