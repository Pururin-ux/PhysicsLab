## PhysicsLab frontend taste rules

Use Taste Skill only as a visual-quality helper, not as a replacement for the PhysicsLab learning model.

PhysicsLab is not a SaaS landing page.
PhysicsLab UI should feel like a beautiful educational instrument:
- clear;
- dense enough;
- responsive;
- physically meaningful;
- non-generic;
- not decorative for its own sake.

Default PhysicsLab visual parameters:
- DESIGN_VARIANCE: 4
- MOTION_INTENSITY: 2
- VISUAL_DENSITY: 7

For simulation/lab scenes:
- prioritize “more experiment in one viewport”;
- avoid one huge card per screen;
- controls and observation should be visible close together;
- do not solve layout problems only by reducing padding/gap/font-size;
- change composition when needed;
- keep graphs readable;
- keep touch targets usable;
- avoid sticky controls unless explicitly requested.

Codex must not self-grade visual quality.
For visual tasks, report:
- screenshot paths;
- first viewport contents;
- element bounding boxes;
- what is above/below the fold;
- what changed in composition.

Human reviewer decides whether visual quality is acceptable.
