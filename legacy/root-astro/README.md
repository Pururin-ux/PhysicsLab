# PhysicsLab

PhysicsLab - интерактивный учебник-тренажер по физике для школьников Беларуси.

Проект помогает разбирать школьную физику через понятные учебные маршруты: формула, график, задача, симуляция, типичная ошибка, проверка понимания и спокойная подготовка к ЦЭ/ЦТ.

## Аудитория

- школьники Беларуси;
- ученики, которым нужна понятная физика без сухого корпоративного EdTech;
- подготовка к школьной программе, ЦЭ/ЦТ и формату РИКЗ;
- фокус на навыках: графики, единицы СИ, выбор закона, проверка ответа, численный ответ.

## Статус проекта

PhysicsLab сейчас находится в переходном состоянии и не является production-продуктом.

В репозитории одновременно есть:

- старый HTML-прототип;
- дизайн-исследования;
- документация по методике и миграции;
- новая Astro foundation для PhysicsLab 2.0.

Выбранное визуальное направление для PhysicsLab 2.0: **Dark Anime Lab**.

## Структура проекта

```text
pages/        старый HTML-прототип, legacy reference
css/          стили старого прототипа
js/           скрипты старого прототипа
assets/       ассеты старого прототипа
prototypes/   статические дизайн-исследования направлений PhysicsLab 2.0
docs/         методика, правила контента, дизайн-решения, планы миграции
src/          новая Astro foundation PhysicsLab 2.0
tests/        Playwright workflow для screenshots и будущих e2e-проверок
artifacts/    generated screenshots и временные результаты проверок, не коммитить
```

## Legacy, Prototype, Foundation

`pages/`, `css/`, `js/`, `assets/` - legacy-прототип. Он нужен как источник идей и reference, но не является финальной архитектурой PhysicsLab 2.0.

`prototypes/` - визуальные исследования. Эти HTML/CSS-страницы помогают сравнивать направления, но не являются production-кодом.

`src/` - новая Astro foundation. Сейчас она показывает направление Dark Anime Lab через страницу `/lab-preview/` и базовые компоненты. Это основа для дальнейшей миграции, а не готовая версия сайта.

## Как запустить Astro preview

Установить зависимости:

```bash
npm install
```

Запустить локальный dev server:

```bash
npm run dev
```

Открыть Astro preview:

```text
http://127.0.0.1:4321/lab-preview/
```

Если Astro выбрал другой порт, смотри адрес в выводе терминала.

## Screenshots workflow

Скриншоты нужны перед и после UI/design-pass задач, чтобы не менять дизайн вслепую.

Запуск:

```bash
npm run screenshots
```

Скриншоты сохраняются в:

```text
artifacts/screenshots/lab-preview/
```

Проверяемые viewport:

- desktop: `1440x1000`
- laptop: `1280x900`
- tablet: `768x1024`
- mobile: `390x844`
- narrow-mobile: `360x800`

Для каждого viewport создаются full-page screenshot, hero screenshot и screenshot блока `#chapter-demo`.

## Что пока не готово к production

- `/lab-preview/` - демо-страница, а не финальная главная страница.
- Физический demo-контент требует проверки автора.
- Старые HTML-страницы еще не мигрированы в Astro/MDX.
- Нет финальной content model для глав, задач, квизов и повторения.
- Нет production-ассетов персонажей и character bible.
- Нет persistence прогресса, аккаунтов, backend или login.
- Нет полноценной библиотеки физической логики и unit tests для нее.
- Нет финального accessibility/a11y pipeline.
- Нет утвержденной production-дизайн-системы, есть foundation выбранного направления.

## Политика языка UI

Видимый пользовательский интерфейс PhysicsLab должен быть на русском языке.

Английский допустим:

- в именах файлов;
- в названиях компонентов;
- в props, типах, CSS-классах и переменных;
- в технических документах, когда это термин стека или кода.

Английский не должен попадать в видимый UI как служебный мусор: `prototype/demo`, `authorReviewRequired`, `FormulaCard`, `Lab cockpit`, `Feedback`, `planned` и похожие строки должны заменяться русскими пользовательскими формулировками.

## Главные правила разработки

- Не превращать PhysicsLab в generic corporate EdTech.
- Не добавлять реальные картинки, мемы, кадры аниме или персонажей без asset policy.
- Не копировать учебники, задания РИКЗ, сборники и чужие формулировки.
- Новый физический контент должен иметь статус проверки.
- Один экран должен держать одну главную учебную мысль.
- Формула, переменные, единицы, график и задача должны быть визуально связаны.
- Mobile-версия считается полноценной, а не остаточной.
